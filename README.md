# 🔭 PJSK TW Observatory｜世界計畫台服排名觀測站

歡迎使用 **PJSK TW Observatory（世界計畫台服排名觀測站）**！這是一個專為手遊《世界計畫 繽紛舞台！ feat. 初音未來》台服玩家打造的全方位數據分析與決策支援平台。本平台整合了即時榜單監控、歷代活動存檔、深度競爭分析以及衝榜資源規劃，旨在為每一位「居民」提供最專業、最直觀的數據支持。

數據來源：[Hi Sekai API](https://hisekai.org) (非官方粉絲製作)

## 🎮 使用者指南 (User Guide)

本平台旨在協助玩家在衝榜過程中做出更明智的決策，並深入了解台服的競爭生態。

### 🚀 核心功能介紹

#### 1. 🔴 現時活動 (Live Event)
**掌握戰場的最前線。**
*   **即時榜單**：同步查詢 Top 100 玩家的當前分數、時速（1H/3H/24H）及平均得分。
*   **動態預測線**：系統根據活動剩餘時間，自動推算出 **「理論安全分數」** 與 **「極限死心線」**，協助您調整追擊策略。
*   **精彩片段 (Highlights)**：快速切換至特定名次（如 T500, T1000）的看板，觀察大部隊的推進節奏。

#### 2. 📈 排名趨勢分析 (Rank Trend)
**跨時空的競爭力對比。**
*   **榜線走勢**：觀察特定名次（如 T100, T1000）在不同活動間的分數變化，掌握台服環境的通膨或緊縮趨勢。
*   **多維篩選**：支援按年份、團體、活動類型進行過濾，找出最具參考價值的歷史數據。

#### 3. 🌐 World Link 深度分析
**針對 WL 特殊機制優化的分析模組。**
*   **日均分校正 (chDavg)**：由於 WL 各章節天數不一，系統會自動進行權重校正，提供公平的「日均競爭強度」比較。
*   **全域基準切換**：可切換 **T100 Base** 或 **T500 Base**，直觀比較各角色章節的熱度差異。

#### 4. 🕵️ 玩家生態結構 U(K) (Player Structure)
**量化排名的流動性與固化程度。**
*   **不重複率分析**：透過 U(K) 曲線判別特定排名區間是「百家爭鳴」還是「名次固化」。
*   **解讀指南**：曲線越陡代表人員流動率高（新血多）；曲線越平緩代表該區間多為固定常客。

#### 5. 🧮 衝榜工具箱 (Resource Estimator)
*   **精確計算機**：輸入目前的單場 PT 與體力計畫，系統會結合歷史數據自動估算所需的「大補充罐」數量與總場數。
*   **玩家狀態查詢**：輸入 ID 即可調閱該玩家的綜合力組成（區域道具、角色等級、家具等）及歌曲通關現況。

---

### 💡 如何使用？

*   **如何進行有效的「衝榜預估」？**
    1. 進入 **「資源計算機」**。
    2. 參考相似性質的歷史活動（如：同團體、同類型）。
    3. 填寫預計的單場得分，系統將為您計算出最接近現實的資源消耗。
*   **圖表判讀小技巧**：
    *   **斜率越陡**：代表名次間的分數差距極大，存在明顯的斷層。
    *   **曲線越平緩**：代表競爭非常激烈，稍有鬆懈名次便會大幅下滑。

## 💻 開發者資訊 (Developer Info)

本專案採用現代化的前端技術棧開發，注重效能與擴充性。

### 🏗️ 系統架構與設計模式
本專案採用 **三層式架構 (Three-Tier Architecture)** 與 **容錯設計 (Fault-Tolerant Design)**：
*   **展示層 (Presentation Layer)**: 基於 React 的 SPA 應用。
*   **服務層 (Service Layer)**: 位於 `src/services/`，負責封裝外部 API 請求與資料處理。
*   **資料層 (Data Layer)**: 整合 Supabase 與 Hi Sekai API。
