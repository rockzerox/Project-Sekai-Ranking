# 🏆 Hi Sekai TW - Project Sekai 台服數據分析與排名監控平台

歡迎使用 **Hi Sekai TW**！這是一個專為手遊《世界計畫 繽紛舞台！ feat. 初音未來》台服玩家打造的全方位數據分析與決策支援平台。本平台整合了即時榜單監控、歷代活動存檔、深度競爭分析以及衝榜資源規劃，旨在為每一位「居民」提供最專業、最直觀的數據支持。

數據來源：[Hi Sekai API](https://hisekai.org) (非官方粉絲製作)

---

## 🎮 使用者指南 (User Guide)

本平台旨在協助玩家在衝榜過程中做出更明智的決策，並深入了解台服的競爭生態。

### 🚀 核心功能介紹

#### 1. 🔴 現時活動 (Live Event)
**掌握戰場的最前線。**
*   **即時榜單**：同步查詢 Top 100 玩家的當前分數、時速（1H/3H/24H）及平均得分。
*   **動態預測線**：系統根據活動剩餘時間，自動推算出 **「理論安全分數」** 與 **「極限死心線」**，協助您調整追擊策略。
*   **精彩片段 (Highlights)**：快速切換至特定名次（如 T500, T1000）的看板，觀察大部隊的推進節奏。

#### 2. 📈 排名趨勢分析 (Rank Trend)
**跨時空的競爭力對比。**
*   **榜線走勢**：觀察特定名次（如 T100, T1000）在不同活動間的分數變化，掌握台服環境的通膨或緊縮趨勢。
*   **多維篩選**：支援按年份、團體、活動類型進行過濾，找出最具參考價值的歷史數據。

#### 3. 🌐 World Link 深度分析
**針對 WL 特殊機制優化的分析模組。**
*   **日均分校正 (chDavg)**：由於 WL 各章節天數不一，系統會自動進行權重校正，提供公平的「日均競爭強度」比較。
*   **全域基準切換**：可切換 **T100 Base** 或 **T500 Base**，直觀比較各角色章節的熱度差異。

#### 4. 🕵️ 玩家生態結構 U(K) (Player Structure)
**量化排名的流動性與固化程度。**
*   **不重複率分析**：透過 U(K) 曲線判別特定排名區間是「百家爭鳴」還是「名次固化」。
*   **解讀指南**：曲線越陡代表人員流動率高（新血多）；曲線越平緩代表該區間多為固定常客。

#### 5. 🧮 衝榜工具箱 (Resource Estimator)
*   **精確計算機**：輸入目前的單場 PT 與體力計畫，系統會結合歷史數據自動估算所需的「大補充罐」數量與總場數。
*   **玩家狀態查詢**：輸入 ID 即可調閱該玩家的綜合力組成（區域道具、角色等級、家具等）及歌曲通關現況。

---

### 💡 如何使用？

*   **如何進行有效的「衝榜預估」？**
    1. 進入 **「資源計算機」**。
    2. 參考相似性質的歷史活動（如：同團體、同類型）。
    3. 填寫預計的單場得分，系統將為您計算出最接近現實的資源消耗。
*   **圖表判讀小技巧**：
    *   **斜率越陡**：代表名次間的分數差距極大，存在明顯的斷層。
    *   **曲線越平緩**：代表競爭非常激烈，稍有鬆懈名次便會大幅下滑。

## 💻 開發者資訊 (Developer Info)

本專案採用現代化的前端技術棧開發，注重效能與擴充性。

### 🛠️ 技術棧 (Tech Stack)
*   **框架**: React 18 (Functional Components, Hooks)
*   **語言**: TypeScript (嚴格型別定義)
*   **建置工具**: Vite (高速開發與打包)
*   **樣式**: Tailwind CSS (Utility-first CSS)
*   **動畫**: Framer Motion (流暢的介面轉場)
*   **圖表**: 自定義 SVG 渲染引擎 (優化大數據展示效能)

### 🏗️ 系統架構與設計模式
*   **Context API 驅動**: 使用 `ConfigContext` 統一管理全域活動設定、顏色映射及 World Link 邏輯。
*   **高效能數據處理**: 
    *   針對大數據分析（如 U(K) 曲線）採用 `useRef` 快取策略與 `useMemo` 計算優化。
    *   封裝 `fetchJsonWithBigInt` 解決 JavaScript 原生 JSON 無法精確處理 64 位元整數 ID 的問題。
*   **響應式設計**: 採用 Mobile-first 策略，針對手機版提供橫向觀看提醒與觸控優化。

### 📂 專案結構
*   `/src/components/pages`: 各功能模組的主要視圖。
*   `/src/contexts`: 全域狀態管理。
*   `/src/hooks`: 封裝 API 請求與複雜邏輯。
*   `/src/utils`: 時間、數學及遊戲邏輯工具函式。

```text
.
├── docs/                   # 專案規格書與技術文件 (Mermaid 序列圖)
├── src/
│   ├── components/
│   │   ├── charts/         # 自定義 SVG 圖表組件
│   │   ├── layout/         # 全域佈局 (Sidebar, Navbar)
│   │   ├── pages/          # 核心功能視圖 (Live, Trend, Analysis, etc.)
│   │   └── ui/             # 原子級 UI 組件 (Card, Button, Tooltip)
│   ├── config/             # 遊戲常數與 UI 文字設定
│   ├── contexts/           # ConfigContext (全域狀態與顏色映射)
│   ├── hooks/              # 業務邏輯與 API 請求封裝 (useRankings, useEventList)
│   ├── services/           # 外部 API 整合邏輯
│   ├── utils/              # 核心演算法 (U(K), WL 校正, 時間處理)
│   ├── App.tsx             # 視圖切換與根邏輯
│   └── index.tsx           # 應用程式進入點
├── package.json            # 依賴管理
└── tailwind.config.js      # 樣式系統配置
```
